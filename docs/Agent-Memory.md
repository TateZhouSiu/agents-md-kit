# Agent Memory

`agent_memory/` is a small persistent state folder for coding agents.

## Files

- `context.md`: stable facts about the project.
- `progress.md`: current task state and next step.
- `bugs.md`: active issues, failed attempts, and repeated pitfalls.
- `archive/`: old compressed notes when a file grows too long.

## Rules

- Read all three files before non-trivial work.
- Update them when a phase completes, direction changes, a failed attempt matters, or a session is about to end.
- Keep only current effective information.
- Archive old content before it becomes noisy.
